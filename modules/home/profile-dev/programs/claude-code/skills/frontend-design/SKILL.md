Create distinctive frontend interfaces that avoid generic "AI slop" aesthetics. Implement working code with exceptional attention to aesthetic details.

## Design Thinking

Before coding, commit to a BOLD aesthetic direction:
- **Purpose**: What problem does this solve? Who uses it?
- **Tone**: Pick an extreme: brutally minimal, maximalist chaos, retro-futuristic, organic/natural, luxury/refined, playful/toy-like, editorial/magazine, brutalist/raw, art deco/geometric, soft/pastel, industrial/utilitarian, etc.
- **Constraints**: Technical requirements (framework, performance, accessibility)
- **Differentiation**: What makes this UNFORGETTABLE?

**CRITICAL**: Choose a clear direction and execute with precision. Intentionality > intensity.

Implement code that is: production-grade, visually striking, cohesive aesthetic, meticulously refined.

## Aesthetics Guidelines

- **Typography**: Distinctive fonts only. No Arial/Inter/Roboto. Pair bold display font with refined body font.
- **Color**: Commit to cohesive aesthetic. CSS variables for consistency. Dominant colors + sharp accents.
- **Motion**: CSS animations for HTML. Motion library for React. Focus on high-impact moments: orchestrated page loads with staggered reveals, scroll-triggering, surprising hover states.
- **Spatial Composition**: Unexpected layouts. Asymmetry, overlap, diagonal flow, grid-breaking, generous negative space OR controlled density.
- **Backgrounds**: Create atmosphere. Gradient meshes, noise textures, geometric patterns, layered transparencies, dramatic shadows, decorative borders, custom cursors, grain overlays.

**NEVER**: Generic fonts (Inter, Roboto, Arial, system fonts), cliched colors (purple gradients on white), predictable layouts, cookie-cutter design.

Vary designs: light/dark themes, different fonts/aesthetics. NEVER converge on common choices (Space Grotesk, etc).

**Match complexity to vision**: Maximalist = elaborate code with extensive animations. Minimalist = restraint, precision, spacing, typography, subtle details.

## リファレンスデザイン

### ターミナル系ポートフォリオ

**こだわり**
- メタファーへの徹底的コミット — 「ターミナル×宇宙」を装飾ではなくアイデンティティとして貫く
- 制約が個性を生む — 標準UIパターンを避け、独自ルールを一貫適用することで記憶に残る
- 個人ブランド=デザインシステム — ビジュアル自体が「クラフトにこだわる開発者」というメッセージ

**グラフィック**
- 配色: 深いネイビー背景、シアンのアクセント+グロー効果
- 文字: ナビ/見出しにモノスペース、本文にサンセリフ
- モチーフ: `└─` ツリーナビ、浮遊パーティクル、ディザリング画像、ドット区切り
- レイアウト: 固定サイドバー + カードセクション + 2カラムグリッド

**ユーザー体験**
- 探索的発見 — ツリー構造が「ディレクトリを探る」感覚を演出
- 世界観への没入 — パーティクルとグローが静的ページに生命感を与える
- 開発者としての共感 — ターミナル風UIが技術者に「仲間」と感じさせる

### 技術コミュニティプラットフォーム

**こだわり**
- コンテンツファースト — デザインは読みやすさに奉仕し、主張しない
- プラットフォーム思考 — 個別ページではなくシステムとして設計（テーマ、スペーシング、コンポーネント）
- 抑制が信頼を生む — 装飾を減らすほどプロフェッショナルに見える

**グラフィック**
- 配色: テーマ対応 (白/ライトグレー、ダークブルー)、青のインタラクティブアクセント
- 文字: Inter (600/700)、明確な階層、広い行間
- 部品: メタデータ付きカード (いいね/ブックマーク)、絵文字バッジ、ミニマルナビ
- レイアウト: カードグリッド、一貫したパディング、レスポンシブ

**ユーザー体験**
- スキャン可能性 — カード+階層で流し読みしやすい、目的の記事にすぐ到達
- 認知負荷の軽減 — 一貫したパターンで「考えずに使える」
- 社会的証明 — いいね/コメント数が質の指標として機能

### シニアエンジニアポートフォリオ

**こだわり**
- サイト自体が実績証明 — 「アクセシブルでピクセルパーフェクト」という専門性をサイトで実証
- 実用性と美の融合 — 技術的洗練さを保ちながらシンプルさを追求
- 段階的情報開示 — フルレジュメ・アーカイブへの導線で情報密度をコントロール

**グラフィック**
- 配色: ダークブルー系ベースのダークテーマ
- 文字: Inter、モダンで高可読性
- モチーフ: イースターエッグ（趣味への言及）でカジュアル要素を添える
- レイアウト: セクション型階層（About→Experience→Projects）、リスト型経歴表示

**ユーザー体験**
- 第一印象で専門性確立 — 職位を早期提示
- 信頼構築 — 大手企業での経歴リスト化で実績を立証
- プロ+親近感 — 趣味・家族言及で人間味を出す

### クリエイティブエンジニアポートフォリオ

**こだわり**
- 芸術作品としてのデジタル体験 — 機能を超えて感情的な物語を伝える
- プロジェクト毎のストーリー性 — 各作品に独自の世界観、記憶に残る体験設計
- 技術とデザインの融合 — 3Dグラフィックス、モーションで没入感創出

**グラフィック**
- 配色: プロジェクト毎に異なるパレット（紫、ピンク、青など）、ダーク背景でコンテンツを引き立て
- 文字: 大胆な見出し配置、モーション付きテキストアニメーション
- モチーフ: 3Dモデル（彫刻など）、グラデーション背景アニメーション
- レイアウト: ヒーロー領域に大型テキスト、スクロールトリガー、グリッドベースのプロジェクト展示

**ユーザー体験**
- 驚嘆と共感 — 技術力と人間味の共存
- 探究心の喚起 — 各プロジェクト詳細への自然な導線
- 直感的位置把握 — セクション毎のビジュアル変化で現在地を認識

### 開発ツールSaaS

**こだわり**
- "Purpose-built" の哲学 — 汎用ではなく「専用設計」という明確なポジショニング
- シンプリシティの追求 — 複雑なプロセスを整理・簡素化
- パフォーマンスへの執着 — preload、フォント最適化、遅延読み込みを徹底

**グラフィック**
- 配色: ほぼ純黒 (#08090a) のダークテーマ、コントラストを抑えた洗練された配色
- 文字: Inter Variable（可変フォント）、体系的なタイポグラフィシステム、text-wrap: balance
- モチーフ: ブラー→シャープのフェードイン、単語単位アニメーション、文字へのグラデーション
- レイアウト: Flexbox中心、変数管理されたスペーサー、レスポンシブ

**ユーザー体験**
- 第一印象: ダークUIで「プロフェッショナル」「技術的洗練」
- 効率性の約束 — 洗練されたUIで「無駄のない、速い体験」を体現
- 信頼+革新 — 堅牢さとモダンさの両立、アニメーションで最先端感を演出
